The Intel® IPP Unified Image Codecs (UIC) included with the Intel® IPP Samples are a set of C++ interfaces that simplify the use of various image codecs in applications. UIC includes abstract base classes to specify particular interfaces and provides implementation of some image codecs, for example JPEG and JPEG2000, built on top of these base classes.

About This Software

The Unified Image Codecs API is intended to standardize interface of various image codecs and to simplify using image codecs in applications. UIC defines specific interface and allows extension through implementing additional functionality in derived classes.

Unified Image Codecs Concepts

This section explains the purpose and structure of the Unified Image Codecs and looks over some of the basic concepts used in UIC implementation. It also describes the supported data formats and operation principles.

Purpose and Basic Features

The Unified Image Codecs are intended to standardize the interface of available image codecs and their applications under development. The purpose is to facilitate reuse of codecs in a transcoding environment, or other similar environments that require code reuse. For example, reusing a specific implementation of a JPEG codec in such environment may require non-trivial code changes. UIC addresses such adaptability and portability issues.

UIC defines abstract encoder and decoder classes according to their functions, and require that every codec supports a specific minimum interface. Thus, the user can write a UIC interface for a media application and expect plug-and-play of various image codecs.

Another aspect of the UIC is a cross-platform/OS capability. UIC provides compatibility of a codec code with a specific OS by abstract interface of input/output streams. To adapt to a new OS, the user can rewrite a small set of input/output streams.

UIC are designed to simplify and facilitate:

  • developing and testing of media application

  • adding new codec support to an application

  • porting an application to another platform/OS.

UIC Structure

Major functionality and features for UIC include the following:

  • Base Classes (base codec, base streams) that serve as a basis for development of derivative classes and provide the level of abstraction needed to integrate codecs and various data sources\destinations (file, network, etc.)

  • Derived Classes (specific image codecs and specific streams) that are derived from base classes to provide specific codec functionality with set of specific data source\destination.

  • Image Format & Raster Data Storage Classes for storing raster image data and representing image formats. The top of these classes is the UIC::Image class. See description of the classes in the Base classes chapter.

Error Reporting

Unified Image Codecs uses two mechanisms to report errors:

  • Status codes, which are returned by functions and specify the cases of successful or failed execution. This mechanism helps to serve the hardest non-recoverable errors, like memory allocation failures or input/output stream failures. The status codes provide minimal information about the failures and in some cases relay on additional information to interpret the true cause (see the next point). Currently, only two status codes are defined: ExcStatusOk and ExcStatusFail. Fail status indicates that the situation is non-recoverable and the coding process cannot continue successfully.

  • Diagnostic stream to receive output from a particular codec to accumulate diagnostic information (as specific codes) during specific encoding\decoding process. It is attached before any sufficient operations to the codec to receive diagnostic data and interpret an unexpected situation.

To use UIC codecs, you should include header files from

  • uic/core/uic/include directory for the UIC core functionality

  • uic/io/uic_io/include directory for the UIC input/output functionality and

  • header files for the codecs to be used in the application from uic/codec/image/<codec>/….

For example, only JPEG codec header files may be included if no other codecs are planned to be used.

To avoid duplicating of the common functionality for an encoder and a decoder, a codec may have a common directory. If in a specific application it is planned to use, for example, only the JPEG decoder, this application should include header files from the following directories:

Usage Examples

Initialization

To initialize the encoder, call the Init() function that allocates internal component objects and sets them to the initial state, and AttachStream() function that attaches the output stream. The stream should be initialized and opened before AttachStream() function is called.

No flags are required for initialization of the encoder. See next example for the initialization example:

Example 1. JPEGEncoder Initialization
ExcStatus       status;
JPEGEncoder     jpegenc;
CStdFileOutput  fo;

if(!BaseStream::IsOk(fo.Open(argv[1])))
    return 1;

status = jpegenc.Init();
if(ExcStatusOk != status)
    return 1;

status = jpegenc.AttachStream(fo);
if(ExcStatusOk != status)
    return 1;

De-Initialization

To de-initialize the encoder, invoke Close() method that frees all allocated memory and sets the remaining parameters to initial values. The destructor invokes Close() method as well. De-initialization does not affect the output bitstream, but it may write diagnostic descriptors to the diagnostic stream in case of a problem.

Example 2. UIC::JPEGEncoder Usage
CStdFileOutput  fout;
if(!BaseStream::IsOk(fout.Open(argv[1])))
    return 1;

Image    imageCn; ExcStatus     status;

// ... skip image data initialization and filling

JPEGEncoder     encoder;

status = encoder.Init();
if(ExcStatusOk != status)
    return 1;

status = encoder.AttachStream(out);
if(ExcStatusOk != status)
    return 1;

status = jpegenc.SetParams(JPEG_BASELINE, JC_RGB, JSS_444, 0, 0, 75);
if(ExcStatusOk != status)
    return 1;

status = encoder.WriteHeader();
if(ExcStatusOk != status)
    return 1;

status = encoder.WriteData();
if(ExcStatusOk != status)
    return 1;

    return 1;

    return 1;

dataOrder.ReAlloc(Interleaved, nOfComponents);
dataOrder.PixelStep()[0] = nOfComponents;
dataOrder.LineStep() [0] = AlignStep(geometry.RefGridRect().Width() *
                                    nOfComponents);

geometry.SetEnumSampling(S444);

imageCn.ColorSpec().ReAlloc(nOfComponents);

JPEG 2000 Coding/Decoding Example

Example 11. JPEG 2000 Example
#include "stdfilein.h"
#include "stdfileout.h"
#include "uic_jp2_enc.h"
#include "uic_jp2_dec.h"

using namespace UIC;


inline uint AlignStep(uint step) { return (step + 31) & 0xFFFFFFE0; }

char* color_str[] = { "Unknown", "Grayscale", "RGB", "BGR", "YCbCr",
                      "CMYK", "YCCK", "YUV" };


ExcStatus encode_jpeg2000( Image& image, BaseStreamOutput& out)
{
    JP2Encoder  jp2enc;

    RET_ON_EXC(jp2enc.Init());
    RET_ON_EXC(jp2enc.AttachStream(out));
    RET_ON_EXC(jp2enc.AttachImage(image));
    RET_ON_EXC(jp2enc.WriteHeader());
    RET_ON_EXC(jp2enc.WriteData());

    return ExcStatusOk;
}

ExcStatus decode_jpeg2000( BaseStreamInput& in, Image&  image)
{
    JP2Decoder  decoder;

    RET_ON_EXC(decoder.Init());
    RET_ON_EXC(decoder.AttachStream(in));

    ImageSamplingGeometry samplingGeometry;
    RET_ON_EXC(decoder.ReadHeader(image.ColorSpec(), samplingGeometry));
    uint nOfComponents = samplingGeometry.NOfComponents();
    ImageDataOrder dataOrder;
    RET_ON_EXC(dataOrder.ReAlloc(Plane, nOfComponents));
    dataOrder.SetDataType(T32s);

    for(uint component = 0; component < nOfComponents; component++)
    {
        dataOrder.PixelStep()[component] = NOfBytes (dataOrder.DataType());
        dataOrder.LineStep() [component] =
                AlignStep(samplingGeometry.RefGridRect().Width() *
                    dataOrder.PixelStep()[component]);
    }
    RET_ON_EXC(image.Buffer().ReAlloc(dataOrder, samplingGeometry));
    RET_ON_EXC(decoder.ReadData(image.Buffer().DataPtr(),
            image.Buffer().BufferFormat().DataOrder()));

    return ExcStatusOk;
}

int main(int argc, char* argv[])
{
    ExcStatus res;
    CStdFileInput fi;
    CStdFileOutput fo;
    Image image;

    if(argc < 2)
        return 1;

    if(!BaseStream::IsOk(fi.Open(argv[1])))
        return 1;

    res = decode_jpeg2000(fi,image);
    if(!IsOk(res))
        return 1;

    printf("file: %s\n",argv[1]);
    printf("%dx%dx%d, %d-bits, color: %s\n",
        image.Buffer().BufferFormat().SamplingGeometry().RefGridRect().Width(),
        image.Buffer().BufferFormat().SamplingGeometry().RefGridRect().Height(),
        image.Buffer().BufferFormat().SamplingGeometry().NOfComponents(),
        image.ColorSpec().DataRange()[0].BitDepth() + 1,
        color_str[image.ColorSpec().EnumColorSpace()]);

    if(!BaseStream::IsOk(fo.Open("out.jp2")))
        return 1;

    res = encode_jpeg2000(image,fo);
    if(!IsOk(res))
        return 1;

    return 0;
}

ImageDataPtr dataPtr;

status = pngdec.Init();
if(ExcStatusOk != status)
    return JPEG_ERR_INTERNAL;

status = pngdec.AttachStream(in);
if(ExcStatusOk != status)
    return JPEG_ERR_INTERNAL;

pngdec.SetByteOrder(param.endianess);

status = pngdec.ReadHeader(colorSpec, geometry);
if(ExcStatusOk != status)
    return JPEG_ERR_INTERNAL;

int nOfComponents = geometry.NOfComponents();

if(colorSpec.DataRange()->BitDepth() + 1 <= 8)
{
    dataOrder.SetDataType(T8u);
    du = sizeof(Ipp8u);
}
else
{
    dataOrder.SetDataType(T16u);
    du = sizeof(Ipp16u);
}

dataOrder.ReAlloc(Interleaved, nOfComponents);
dataOrder.PixelStep()[0] = nOfComponents;
dataOrder.LineStep()[0] = ((geometry.RefGridRect().Width() *
                            nOfComponents + 3) & ~3)*du;

imageCn.ColorSpec().ReAlloc(nOfComponents);
imageCn.ColorSpec().SetColorSpecMethod(Enumerated);
imageCn.ColorSpec().SetComponentToColorMap(Direct);

for(int i = 0; i < nOfComponents; i++)
{
    if(colorSpec.DataRange()->BitDepth() + 1 <= 8)
    {
        imageCn.ColorSpec().DataRange()[i].SetAsRange8u((1 <<
                (colorSpec.DataRange()->BitDepth() + 1)) - 1);
    }
    else
    {
        imageCn.ColorSpec().DataRange()[i].SetAsRange16u((1 <<
                (colorSpec.DataRange()->BitDepth() + 1)) - 1);
    }
}

ImageEnumColorSpace in_color; ImageEnumColorSpace out_color;
in_color = colorSpec.EnumColorSpace();

switch(nOfComponents)
{
    case 1:     out_color = (in_color == Unknown) ? Unknown : Grayscale; break;
    case 3:     out_color = (in_color == Unknown) ? Unknown : RGB;      break;
    case 4:     out_color = (in_color == Unknown) ? Unknown : RGBA;     break;
    default: out_color = Unknown;       break;
}

imageCn.ColorSpec().SetEnumColorSpace(out_color);

int r;
IppiSize size;

size.width      = geometry.RefGridRect().Width();
size.height = geometry.RefGridRect().Height();

r = image.Alloc(size, nOfComponents, colorSpec.DataRange()->BitDepth() + 1, 1);

if(0 != r)
    return JPEG_ERR_ALLOC;

image.Color(GetColor(out_color));

if(image.Precision() <= 8)
{
    dataPtr.p8u = image;
}
else
{
    dataPtr.p16s = image;
}

param.sampling = image.Sampling();

    return JPEG_ERR_INTERNAL;

Features of Component Behavior With Respect to Input Data
